## Tầm nhìn

> *Protocols là ngôn ngữ của các agent. Chúng quyết định agent nói gì, nghe gì, đồng ý điều gì và từ chối điều gì. Lộ trình này đưa bạn từ những thông điệp đầu tiên đến kiến trúc giao tiếp đa tác nhân chuẩn công nghiệp — bảo mật, quan sát được và mở rộng được.*

## Khóa 1 — MAS Bootcamp

**Repo**: [aap-bootcamp](https://github.com/CocAgent/aap-bootcamp) · 10 tuần · Nhập môn

Điểm khởi đầu cho người chưa có kinh nghiệm về hệ đa tác nhân. Không cần background AI — chỉ cần Python cơ bản. Mục tiêu: xây dựng và chạy được agent đầu tiên trong 2 giờ đầu tiên.

**Nội dung cốt lõi:**

| Tuần | Chủ đề | Bạn làm được |
|------|--------|-------------|
| 01 | Agent là gì? Tại sao cần nhiều agent? | Phân biệt agent / program / microservice |
| 02 | Agent đơn giản bằng Python | `class Agent` với `perceive → decide → act` |
| 03 | Tool use & function calling | Agent gọi API bên ngoài, parse JSON |
| 04 | Hai agent nói chuyện | Request–Response qua REST |
| 05 | Bộ nhớ và trạng thái | Stateful agent, session management |
| 06 | Agent với LLM | Tích hợp OpenAI/Anthropic API |
| 07 | Workflow đơn giản | Pipeline 3 agent tuần tự |
| 08 | Xử lý lỗi và retry | Circuit breaker, timeout, fallback |
| 09 | Kiểm thử agent | pytest, mock, behavioral testing |
| 10 | Dự án Bootcamp | End-to-end agent system nhỏ |

**Bạn sẽ xây dựng:** Agent Python từ đầu · Workflow 3 tác nhân · Tích hợp LLM API · Bộ test 20 trường hợp

**Điều kiện tiên quyết:** Python 3.x cơ bản (list, dict, class). Không cần AI/ML.

| 11 | Kiểm thử giao tiếp | Mock transport, async test |
| 12 | Hiệu năng messaging | Throughput, latency benchmark |
| 13 | Dự án: Hệ thống thông báo đa tác nhân | Notification hub + 5 subscriber agents |

**Bạn sẽ xây dựng:** FIPA-ACL library (15 performatives) · SPADE multi-agent system · Async messaging với Redis · Bộ test 35 trường hợp

## Khóa 3 — MAS Protocols

Giao thức là tập hợp quy tắc điều phối nhiều agent hoàn thành một tác vụ chung. Khóa này triển khai các giao thức chuẩn từ học thuật và công nghiệp: Contract Net, auction, voting, REST/gRPC.

**Nội dung cốt lõi:**

| Tuần | Chủ đề | Triển khai |
|------|--------|-----------|
| 01 | Giao thức là gì? Phân loại | Taxonomy: negotiation / allocation / consensus |
| 02 | FIPA Interaction Protocols tổng quan | 9 FIPA IPs: Request, Subscribe, Propose... |
| 03 | Contract Net Protocol đầy đủ | CNP 5 pha: CFP→PROPOSE→ACCEPT→EXECUTE→INFORM |
| 04 | Iterated Contract Net | Multi-round, relaxed criteria |
| 05 | Auction protocols | English, Dutch, Vickrey — so sánh |
| 06 | Voting protocols | Borda, Condorcet, Plurality |
| 07 | Blackboard protocol | Shared workspace pattern |
| 08 | REST cho agent API | FastAPI agent endpoint, OpenAPI schema |
| 09 | gRPC cho agent communication | Protobuf schema, bidirectional streaming |
| 10 | Xác thực & phân quyền | OAuth2, API key, agent identity |
| 11 | Giao thức kết hợp | CNP + auction hybrid |
| 12 | Kiểm thử giao thức | Contract test, Pact, property-based |
| 13 | Dự án: Marketplace đa tác nhân | Buyer + Seller + Auctioneer + Registry |

**Bạn sẽ xây dựng:** CNP đầy đủ · Auction simulator · FastAPI agent API · gRPC agent service · Marketplace 4 loại tác nhân

## Khóa 4 — Modern Agent Protocols

**Repo**: [aap-modern](https://github.com/CocAgent/aap-modern) · 13 tuần · Trung cấp+

Kỷ nguyên LLM tạo ra thế hệ giao thức mới: A2A (Agent-to-Agent), MCP (Model Context Protocol), ANP (Agent Network Protocol). Khóa này là cầu nối giữa giao thức cổ điển và hạ tầng AI hiện đại.

**Nội dung cốt lõi:**

| Tuần | Chủ đề | Triển khai |
|------|--------|-----------|
| 01 | Tại sao cần giao thức mới? | So sánh FIPA-ACL vs A2A vs MCP |
| 02 | OpenAI Function Calling spec | Schema chuẩn, tool calling patterns |
| 03 | Anthropic Tool Use Protocol | Claude tool_use vs OpenAI function calling |
| 04 | Model Context Protocol (MCP) | MCP server, MCP client, resource types |
| 05 | Agent-to-Agent (A2A) Protocol | Google A2A spec: agent card, task, artifact |
| 06 | Agent Network Protocol (ANP) | Decentralized agent discovery, DID |
| 07 | Streaming protocols | SSE, WebSocket cho real-time agent output |
| 08 | Long-running tasks | AsyncTask, polling, webhook callback |
| 09 | Multi-modal messaging | Text + image + file trong một thông điệp |
| 10 | Protocol negotiation | Content-type negotiation, capability exchange |
| 11 | Interoperability | Chuyển đổi giữa FIPA ↔ A2A ↔ OpenAI |
| 12 | Protocol versioning | Backward compat, migration strategy |
| 13 | Dự án: Multi-protocol Agent Hub | Hub chuyển đổi giữa 3 protocol stacks |

**Bạn sẽ xây dựng:** MCP server + client · A2A agent card · Protocol bridge (FIPA↔A2A) · SSE streaming agent · Multi-protocol hub

## Khóa 5 — MAS Coordination

**Repo**: [aap-coordination](https://github.com/CocAgent/aap-coordination) · 13 tuần · Nâng cao

Khi nhiều agent phải đưa ra quyết định nhất quán trong môi trường không đáng tin cậy, cần các thuật toán phối hợp phân tán. Khóa này đi từ lý thuyết đồng thuận đến triển khai thực tế chịu lỗi.

**Nội dung cốt lõi:**

| Tuần | Chủ đề | Triển khai |
|------|--------|-----------|
| 01 | Bài toán phối hợp phân tán | CAP theorem, consistency models |
| 02 | Mutual exclusion | Token ring, Ricart-Agrawala |
| 03 | Leader election | Bully algorithm, Ring algorithm |
| 04 | Clock synchronization | Lamport clocks, vector clocks |
| 05 | Consensus cơ bản | Two-phase commit (2PC) |
| 06 | Raft consensus | Leader election + log replication |
| 07 | Byzantine fault tolerance | PBFT overview, trust assumptions |
| 08 | Distributed coordination với ZooKeeper | Watch, ephemeral node, lock service |
| 09 | Deadlock detection & prevention | Wait-for graph, timeout strategy |
| 10 | Load balancing giữa agents | Round-robin, weighted, consistent hashing |
| 11 | Task migration & rebalancing | Graceful agent handoff |
| 12 | Kiểm thử phân tán | Chaos engineering, partition testing |
| 13 | Dự án: Hệ thống phối hợp chịu lỗi | 5-node cluster với Raft + failure scenarios |

**Bạn sẽ xây dựng:** Leader election · Raft log replication · Distributed lock với ZooKeeper · Fault injection test suite · 5-node cluster demo

## Tiêu chuẩn chất lượng

**Triển khai được** — Mỗi dự án cuối khóa phải có `docker compose up` một lệnh. Không có "runs on my machine."

**Quan sát được** — Từ P6 trở đi: mọi agent có health endpoint, structured logging, metrics endpoint.

**Tài liệu gốc** — FIPA spec, Google A2A spec, Anthropic MCP spec, Kubernetes docs. Không phụ thuộc blog thứ cấp.
