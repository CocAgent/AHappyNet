## Tầm nhìn

> *Xây dựng lộ trình học lập trình và AI **có cấu trúc**, **thực hành trước lý thuyết**, và **liên kết chặt chẽ** giữa các khóa — để người học không chỉ biết viết code, mà biết tại sao mình viết như vậy, và cuối cùng có thể tự đặt câu hỏi nghiên cứu và trả lời chúng bằng hệ thống sản xuất thực sự.*

Đây là repo trung tâm điều phối toàn bộ hệ sinh thái khóa học của **CocAgent**. Mỗi chương trình là một lộ trình độc lập nhưng được thiết kế để bổ trợ cho nhau.

## Chọn điểm bắt đầu

```
Bạn là...                                            Bắt đầu từ
──────────────────────────────────────────────────────────────────────────
Hoàn toàn mới với lập trình                    →    python-journey
Biết Python, muốn chuyên nghiệp hoá            →    python-mastery
Muốn vào Data Science / AI                     →    python-journey → mastery → data
Muốn vào DevOps / SRE / Automation             →    python-journey → mastery → automation
Muốn phỏng vấn kỹ thuật (Python)               →    python-journey → mastery → dsa-python
Học C++ từ đầu                                 →    cpp-foundation
Biết C++11/14, muốn Modern C++                 →    cpp-development
Muốn phỏng vấn kỹ thuật (C++)                  →    cpp-foundation → cpp-development → dsa-cpp
Mới với hệ đa tác nhân, muốn bắt đầu nhanh    →    mas-bootcamp (AAP P1)
Muốn hiểu sâu lý thuyết AI agent               →    mas-foundations (MAS K1)
Biết Python, muốn học MARL                     →    mas-marl (MAS K3)
LLM engineer muốn build production MAS         →    mas-llm-agents → mas-capstone
Muốn học giao thức A2A / MCP hiện đại          →    modern-agent-protocols (AAP P4)
Muốn deploy agent lên Kubernetes               →    mas-production (AAP P6)
Muốn nghiên cứu AI / xuất bản paper            →    MAS K1–K4 → mas-capstone
```

## Triết lý chung

Tất cả khóa học trong hệ sinh thái này chia sẻ các nguyên tắc cốt lõi:

**1. Hiểu trước khi dùng** — Mỗi tính năng được giới thiệu kèm *tại sao* nó tồn tại và *vấn đề gì* nó giải quyết.

**2. Code đúng hơn code chạy** — "Chạy được" là điều kiện cần, không phải đủ. Mỗi bài đều có tiêu chí chất lượng rõ ràng.

**3. Kiểm thử không phải tùy chọn** — pytest (Python/MAS/AAP) và GoogleTest (C++) được tích hợp từ giữa lộ trình.

**4. Tài liệu gốc là nguồn chân lý** — FIPA spec, A2A spec, MCP spec, Kubernetes docs, arXiv papers — không phụ thuộc tutorial thứ cấp.

**5. Sản xuất trước notebook** *(MAS và AAP)* — Mọi hệ thống có thể triển khai bằng `docker compose up`. Không có "runs on my machine."

**6. Quan sát được là bắt buộc** *(AAP P6, MAS K5)* — Mọi hệ thống sản xuất có health check, structured logging, metrics endpoint.
